# Contributing

Thank you for improving Soundserve.

## Pull Request Checklist

- Keep changes focused and documented.
- Add or update shared types when API contracts change.
- Run linting, typechecking, and builds locally.
- Include screenshots for visible UI changes.
- Never commit secrets, access tokens, production keys, or private media.

## Branching

Use short descriptive branch names such as `feature/ai-playlist-builder` or `fix/player-queue-order`.
